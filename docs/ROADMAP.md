## 版本命名规范

- **主版本号 (Major)**: 架构重大变更
- **次版本号 (Minor)**: 功能新增
- **修订号 (Patch)**: Bug 修复和小改进

示例:
- `v0.1.0` → `v0.1.1`: 修复 Bug
- `v0.1.0` → `v0.2.0`: 新增功能
- `v0.9.0` → `v1.0.0`: 正式发布
