# MC-Agent-Kit 项目原则

## 核心原则

### 1. 渐进式迭代开发
- 每次迭代聚焦有限目标
- 迭代前阅读文档，理解愿景
- 迭代后总结记录，持续改进
- 小步快跑，频繁交付

### 2. 测试驱动开发
- 所有 Python 功能代码必须编写测试
- 提交前确保所有测试通过
- 测试覆盖率作为质量指标
- 测试即文档

### 3. 文档先行
- 重要决策先写文档
- 代码变更同步更新文档
- 文档与代码保持一致
- 文档是项目的一部分

### 4. 工作空间隔离
- 仅在 `E:\develop\MC-Agent-Kit` 目录工作
- `resources/` 目录只读，不修改
- `resources/` 不上传 git
- 敏感信息不入库

---

## 技术规范

### Python 规范
- Python 版本: 3.13
- 包管理: uv
- 代码风格: 遵循 PEP 8
- 类型注解: 推荐使用

### 项目结构规范
```
MC-Agent-Kit/
├── docs/           # 文档 (必须维护)
├── src/            # 源代码
│   ├── mc_agent_kit/
│   └── tests/      # 测试 (必须编写)
├── resources/      # 参考资料 (只读)
└── pyproject.toml  # 项目配置
```

### Git 提交规范
- 提交信息清晰描述变更
- 每次提交对应一个功能点
- 提交前运行测试
- 推送到远程仓库

---

## 迭代流程

### 迭代前
1. 阅读 `docs/` 下的文档
2. 理解项目整体愿景
3. 查看 `NEXT_ITERATION.md`
4. 规划当前迭代任务

### 迭代中
1. 编写功能代码
2. 编写测试用例
3. 运行测试确保通过
4. 更新相关文档

### 迭代后
1. 更新 `ITERATIONS.md`
2. 更新 `NEXT_ITERATION.md`
3. Commit 并 Push 到 GitHub
4. 生成迭代报告
5. 通过飞书发送报告

---

## 代码质量标准

### 必须遵守
- ✅ 所有功能代码有测试
- ✅ 测试全部通过
- ✅ 代码有类型注解
- ✅ 关键逻辑有注释

### 推荐遵守
- 📝 函数有 docstring
- 📝 模块有说明
- 📝 复杂逻辑有注释

### 禁止行为
- ❌ 提交未测试代码
- ❌ 提交失败测试
- ❌ 硬编码敏感信息
- ❌ 修改 resources 目录

---

## 与 ModSDK 的边界

### 本项目是
- 开发辅助工具
- AI Agent 能力扩展
- 自动化测试调试

### 本项目不是
- ModSDK 本身
- 游戏内运行代码
- Python 2.7 代码

### 版本区分
| 环境 | Python | 运行位置 |
|------|--------|----------|
| ModSDK | 2.7 | 游戏进程 |
| MC-Agent-Kit | 3.13 | 开发机器 |

## 沟通与反馈

### 迭代报告
- 每次迭代完成后生成
- 内容包括：完成内容、遇到问题、下一步计划
- 通过飞书发送给项目负责人

### 问题处理
- 遇到阻塞问题及时记录
- 在迭代报告中说明
- 寻求帮助或调整计划

---

## 安全原则

### 敏感信息
- 用户配置文件不入库
- 账号密码通过环境变量管理
- API Key 不硬编码

### 代码安全
- 不执行不可信代码
- 文件操作需验证路径
- 外部输入需验证
